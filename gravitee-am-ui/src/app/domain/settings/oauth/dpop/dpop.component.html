DPoP

Demonstrating Proof-of-Possession (DPoP, RFC 9449) binds an access token to a key held by the client, so a stolen token cannot be replayed without the corresponding private key.

Require DPoP for all clients

When enabled, this domain-wide floor forces every client to present a DPoP proof at the token endpoint. It cannot be overridden by an application whose own DPoP-bound access tokens setting is off.

Allowed signing algorithms

Optionally restrict which JWS algorithms are accepted in DPoP proofs. Leaving the list empty advertises and accepts all supported algorithms.
